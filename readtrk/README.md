# A Read Track Command Example

This program exits with a lost data flag in the 1793 controller status reg. 

It is possible that when executing a READTRACK command, data bytes may arrive
in back-to-back bursts due to the PLL resyncing between write-gate transitions?
If so then the bytes lost would be in the gaps.

Note that I wrote 1024 (2 sectors) of AA bytes onto track 1 of this disk before running this test.
